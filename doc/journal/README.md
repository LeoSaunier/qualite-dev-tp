# Journal de bord

Ce journal centralise:
- Les décisions architecturales et techniques importantes
- Les réponses aux questions du TP pour l'évaluation

## Décisions

Format recommandé:
- Date: YYYY-MM-DD
- Équipe / Auteurs:
- Sujet:
- Contexte:
- Décision:
- Alternatives considérées:
- Impact (modules, contrats, déploiement):
- Actions:

Exemple:
- Date: 2026-01-11
- Équipe: Command/Write
- Sujet: Stratégie de branchement
- Contexte: Collaboration multi-équipes sur monorepo
- Décision: Trunk-Based + branches de release
- Alternatives: GitFlow (trop lourd)
- Impact: Processus de release simplifié
- Actions: Mise à jour README & CONTRIBUTING

- Exercice 3 / Tâche 5: (release 0.1.0, tag, notes)
